Sistema de layout estilo Ionic con contenedores, secciones, divisores, scroll utilities y estructura de app completa.

Estructura de App (CSS Grid)

Estructura basica con header sticky, content scrollable y footer sticky. Usa CSS Grid con grid-template-rows: auto 1fr auto para calcular automaticamente el espacio.

Demo Interactivo

Activa/desactiva header y footer para ver como el content se adapta automaticamente.

Como Funciona

.ux-page usa CSS Grid:
display: grid;
grid-template-rows: auto 1fr auto;

auto = tamano natural del elemento (0 si no existe)
1fr = todo el espacio restante

.ux-header y .ux-footer son sticky por defecto.

Split Pane (iPad/Tablet Layout)

Layout de dos columnas estilo Ionic. En pantallas pequenas funciona como overlay (drawer), en pantallas grandes >= 992px se muestra como push (redimensiona el contenido).

Uso Basico

Modos de Funcionamiento

mode: 'auto' - (Default) Overlay en mobile, push en desktop
mode: 'push' - Siempre push (redimensiona contenido)
mode: 'overlay' - Siempre overlay (drawer sobre contenido)
mode: 'visible' - Siempre visible en ambos tamanos

Mejores Practicas HTMX + Django

  • hx-target - Apunta a contenedores con ID para reemplazar contenido especifico
  • hx-swap="innerHTML" - Reemplaza el contenido interno sin afectar el contenedor
  • hx-push-url="true" - Actualiza la URL para navegacion SPA-like
  • hx-indicator - Muestra un spinner durante la carga
  • hx-trigger="revealed" - Carga cuando el elemento es visible (infinite scroll)
  • HX-Trigger header - Dispara eventos desde el servidor
  • Detecta peticiones HTMX con request.headers.get('HX-Request')
  • Usa templates parciales para respuestas HTMX (sin base template)
