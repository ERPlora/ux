Por que usar Modular Imports

El bundle completo de UX es de ~1.1MB. Para la mayoria de proyectos, solo necesitas una fraccion de los componentes. Con las importaciones modulares, puedes reducir el tamano entre 70-95%.

Core (Requerido)

El archivo core.min.css contiene todo lo esencial: variables CSS, reset, utilidades de flexbox, spacing, colores, glass morphism, y mas. Es requerido para usar cualquier bundle o componente.

<!-- Core (~85KB) - SIEMPRE requerido -->
<link rel="stylesheet" href="dist/core.min.css">

Que incluye Core:

  • Variables CSS (colores, espaciado, tipografia, etc.)
  • Reset y estilos base
  • Utilidades: .ux-flex, .ux-grid, .ux-p-*, .ux-m-*, etc.
  • Glass morphism: .ux-glass
  • Colores: .ux-bg-*, .ux-text-*, .ux-color-*
  • Posicionamiento: .ux-pos-center, .ux-pos-bottom, etc.
  • Animaciones y transiciones
  • Dark mode y temas

Opcion 1: Core + Bundles Tematicos

Los bundles agrupan componentes relacionados. Ideal cuando necesitas varios componentes de una categoria.

Bundles disponibles:

basic (~84 KB) forms (~172 KB) navigation (~100 KB) overlays (~88 KB) data (~168 KB) feedback (~60 KB) media (~84 KB) layout (~68 KB) pos (~96 KB) hr (~96 KB) manufacturing (~88 KB) utils (~36 KB)
Tip: Los bundles no tienen dependencias entre si. Puedes combinar cualquier cantidad de bundles segun tus necesidades.

Opcion 2: Core + Componentes Individuales

Para maximo control, importa solo los componentes especificos que usas. Cada componente pesa entre 2-10 KB.

Nota: Los archivos de componentes individuales se generan con npm run build:modular -- --components. Por defecto, solo se generan core y bundles.

Uso con npm/bundler

Si usas Vite, Webpack, o similar, puedes importar directamente desde el paquete.

# Instalar
npm install ux

Con SCSS (para customizacion):

// Importar SCSS para customizar variables
@use 'ux/scss' with (
  $ux-primary: #8B5CF6,
  $ux-border-radius: 12px
);

// O importar modulos individuales
@use 'ux/scss/core';
@use 'ux/scss/bundles/forms';

Generar Archivos

Los archivos modulares se generan con el script de build:

# Generar core + todos los bundles
npm run build:modular

# Generar tambien componentes individuales (mas lento)
npm run build:modular -- --components

# Solo el bundle completo (como antes)
npm run build
