¿Por qué usar Modular Imports?

El bundle completo de UX es de ~1.1MB. Para la mayoría de proyectos, solo necesitas una fracción de los componentes. Con las importaciones modulares, puedes reducir el tamaño entre 70-95%.

Core (Requerido)

El archivo core.min.css contiene todo lo esencial: variables CSS, reset, utilidades de flexbox, spacing, colores, glass morphism, y más. Es requerido para usar cualquier bundle o componente.

<!-- Core (~83 KB) - SIEMPRE requerido -->
<link rel="stylesheet" href="dist/core.min.css">

Qué incluye Core:

  • Variables CSS (colores, espaciado, tipografía, etc.)
  • Reset y estilos base
  • Utilidades: .ux-flex, .ux-grid, .ux-p-*, .ux-m-*, etc.
  • Glass morphism: .ux-glass
  • Colores: .ux-bg-*, .ux-text-*, .ux-color-*
  • Posicionamiento: .ux-pos-center, .ux-pos-bottom, etc.
  • Animaciones y transiciones
  • Dark mode y temas

Opción 1: Core + Bundles Temáticos

Los bundles agrupan componentes relacionados. Ideal cuando necesitas varios componentes de una categoría.

Bundles disponibles:

basic (~78 KB) forms (~185 KB) navigation (~116 KB) overlays (~97 KB) data (~140 KB) feedback (~62 KB) media (~91 KB) layout (~144 KB) pos (~114 KB) hr (~94 KB) manufacturing (~106 KB) utils (~21 KB)
Tip: Los bundles no tienen dependencias entre sí. Puedes combinar cualquier cantidad de bundles según tus necesidades.

Opción 2: Core + Componentes Individuales

Para máximo control, importa solo los componentes específicos que usas. Cada componente pesa entre 2-10 KB.

Nota: Los archivos de componentes individuales se generan con npm run build:modular -- --components. Por defecto, solo se generan core y bundles.

Uso con npm/bundler

Si usas Vite, Webpack, o similar, puedes importar directamente desde el paquete.

# Instalar
npm install ux
// Import en tu bundler (Vite, Webpack, etc.)
import 'ux/core';
import 'ux/bundles/forms';
import 'ux/bundles/overlays';

// O componentes individuales
import 'ux/components/button.css';
import 'ux/components/modal.css';

Con SCSS (para personalización):

// Importar SCSS para personalizar variables
@use 'ux/scss' with (
  $ux-primary: #8B5CF6,
  $ux-border-radius: 12px
);

// O importar módulos individuales
@use 'ux/scss/core';
@use 'ux/scss/bundles/forms';

Generar Archivos

Los archivos modulares se generan con el script de build:

# Generar core + todos los bundles
npm run build:modular

# Generar también componentes individuales (más lento)
npm run build:modular -- --components

# Solo el bundle completo (como antes)
npm run build
