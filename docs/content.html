Como Funciona

.ux-page usa CSS Grid:
display: grid;
grid-template-rows: auto 1fr auto;

auto = tamaño natural del elemento (0 si no existe)
1fr = todo el espacio restante

.ux-header y .ux-footer son sticky por defecto.

Modos de Funcionamiento

mode: 'auto' - (Default) Overlay en mobile, push en desktop
mode: 'push' - Siempre push (redimensiona contenido)
mode: 'overlay' - Siempre overlay (drawer sobre contenido)
mode: 'visible' - Siempre visible en ambos tamanos 
<!-- Siempre como drawer (overlay) -->
<div x-data="uxSplitPane({ mode: 'overlay' })" :class="containerClasses">
  ...
</div>

<!-- Siempre visible (sin toggle) -->
<div x-data="uxSplitPane({ mode: 'visible' })" :class="containerClasses">
  ...
</div>

Sidebar a la Derecha

  <aside class="ux-split-pane__side">...</aside>

Alpine.js API: uxSplitPane(config)

x-data="uxSplitPane({
  mode: 'auto',        // 'auto' | 'push' | 'overlay' | 'visible'
  side: 'start',       // 'start' | 'end'
  collapsed: false,    // Iniciar colapsado en desktop
  breakpoint: 992,     // Breakpoint para push mode
  closeOnBackdrop: true,
  closeOnEscape: true
})"

// Propiedades
isOpen       // Boolean - si esta abierto (overlay mode)
isCollapsed  // Boolean - si esta colapsado (desktop)
isVisible    // Boolean - si el sidebar es visible
isOverlay    // Boolean - si esta en modo overlay

// Metodos
open()           // Abrir sidebar
close()          // Cerrar sidebar
toggle()         // Alternar open/close
collapse()       // Colapsar en desktop
expand()         // Expandir en desktop
toggleCollapse() // Alternar collapse

// Computed
containerClasses  // Objeto de clases para :class

// Eventos
@splitpane-open      // Al abrir
@splitpane-close     // Al cerrar
@splitpane-collapse  // Al colapsar
@splitpane-expand    // Al expandir
@splitpane-breakpoint="e => e.detail.isLargeScreen" // Al cambiar de breakpoint

CSS Variables

:root {
  --ux-split-pane-side-width: 270px;     /* Ancho del sidebar */
  --ux-split-pane-side-min-width: 270px; /* Minimo en desktop */
  --ux-split-pane-side-max-width: 28%;   /* Maximo en desktop */
  --ux-split-pane-breakpoint: 992px;     /* Breakpoint push/overlay */
}

Alpine.js Components

Componentes interactivos disponibles.

uxContent(config)

Content con API de scroll completa estilo Ionic.

x-data="uxContent({ scrollEvents: true })"

// Propiedades reactivas
scrollTop       // Posicion vertical actual
scrollLeft      // Posicion horizontal actual
scrollHeight    // Altura total del contenido
clientHeight    // Altura visible
isScrolling     // Boolean - si esta scrolling
isAtTop         // Boolean - si esta en top
isAtBottom      // Boolean - si esta en bottom
scrollProgress  // Number 0-1 - progreso del scroll

// Metodos
scrollToTop(duration)              // Scroll al inicio
scrollToBottom(duration)           // Scroll al final
scrollToPoint(x, y, duration)      // Scroll a punto
scrollByPoint(x, y, duration)      // Scroll relativo
scrollToElement(selector, duration) // Scroll a elemento
getScrollElement()                 // Obtener elemento

// Eventos (si scrollEvents: true)
@ux-scroll="..."        // Cada scroll
@ux-scroll-start="..."  // Inicio de scroll
@ux-scroll-end="..."    // Fin de scroll

Pagina con manejo de ciclo de vida.

x-data="uxPage({ isActive: true })"

// Propiedades
pageId     // ID unico de pagina
isActive   // Boolean - si esta activa

// Metodos
activate()    // Activar pagina
deactivate()  // Desactivar pagina
onEnter()     // Callback de entrada
onLeave()     // Callback de salida

// Eventos
@ux-page-enter="..."  // Al entrar
@ux-page-leave="..."  // Al salir

         hx-target="#posts-list"
         hx-swap="beforeend"
         hx-indicator="#load-spinner">

<!-- Django view -->
# views.py
from django.core.paginator import Paginator

def posts(request):
    page = request.GET.get('page', 1)
    posts = Post.objects.all()
    paginator = Paginator(posts, 10)
    page_obj = paginator.get_page(page)

    # Si es peticion HTMX, solo devolver los items
    if request.headers.get('HX-Request'):
        return render(request, 'partials/posts_items.html', {
            'posts': page_obj,
            'has_next': page_obj.has_next(),
            'next_page': page_obj.next_page_number() if page_obj.has_next() else None
        })

    return render(request, 'posts.html', {
        'posts': page_obj,
        'has_next': page_obj.has_next(),
        'next_page': page_obj.next_page_number() if page_obj.has_next() else None
    })

Mejores Practicas HTMX + Django

  • hx-target - Apunta a contenedores con ID para reemplazar contenido especifico
  • hx-swap="innerHTML" - Reemplaza el contenido interno sin afectar el contenedor
  • hx-push-url="true" - Actualiza la URL para navegacion SPA-like
  • hx-indicator - Muestra un spinner durante la carga
  • hx-trigger="revealed" - Carga cuando el elemento es visible (infinite scroll)
  • HX-Trigger header - Dispara eventos desde el servidor
  • Detecta peticiones HTMX con request.headers.get('HX-Request')
  • Usa templates parciales para respuestas HTMX (sin base template)
