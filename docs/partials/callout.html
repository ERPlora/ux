HTMX + Django

Ejemplos de callouts dinamicos con HTMX y Django para mostrar mensajes de estado, validaciones y notificaciones.

Mensajes de formulario

Mostrar callouts despues de enviar un formulario via HTMX.

Guardado exitoso

Los cambios se han guardado correctamente.

<!-- Formulario con HTMX -->
<form hx-post="/api/save/"
      hx-target="#callout-container"
      hx-swap="innerHTML">
  <input type="text" name="title" class="ux-input">
  <button type="submit" class="ux-button ux-color-primary">Guardar</button>
</form>

<div id="callout-container"></div>
<!-- Django template: partials/callout_success.html -->
<div class="ux-callout ux-callout--success ux-callout--dismissible"
     x-data="uxCallout()" x-show="visible" x-transition>
  <span class="ux-callout__icon">
    <svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 24 24" fill="none"
         stroke="currentColor" stroke-width="2">
      <path d="M22 11.08V12a10 10 0 1 1-5.93-9.14"/>
      <polyline points="22 4 12 14.01 9 11.01"/>
    </svg>
  </span>
  <div class="ux-callout__content">
    <p class="ux-callout__title">{{ title }}</p>
    <p class="ux-callout__text">{{ message }}</p>
  </div>

Validacion en tiempo real

Mostrar errores de validacion mientras el usuario escribe.

El email ya esta registrado en el sistema.

<!-- Input con validacion HTMX -->
<div class="ux-input">
  <input type="email" name="email"
         hx-post="/api/validate-email/"
         hx-trigger="blur, change delay:500ms"
         hx-target="#email-validation">
  <label>Email</label>
</div>
<div id="email-validation"></div>
# Django view
from django.http import HttpResponse
from django.template.loader import render_to_string

def validate_email(request):
    email = request.POST.get('email', '')

    if User.objects.filter(email=email).exists():
        return HttpResponse(render_to_string(
            'partials/callout_error.html',
            {'message': 'El email ya esta registrado en el sistema.'}
        ))

    return HttpResponse('')  # Sin errores

Avisos del sistema

Callouts que se cargan desde el servidor al iniciar la pagina.

Mantenimiento programado

El sistema estara en mantenimiento el 15 de enero de 10:00 a 12:00.

<!-- Cargar avisos al iniciar -->
<div hx-get="/api/system-notices/"
     hx-trigger="load"
     hx-swap="innerHTML"></div>
# Django view
def system_notices(request):
    notices = SystemNotice.objects.filter(
        active=True,
        start_date__lte=timezone.now(),
        end_date__gte=timezone.now()
    )

    return render(request, 'partials/system_notices.html', {
        'notices': notices
    })
<!-- Django template: partials/system_notices.html -->

    {% include "icons/"|add:notice.icon|add:".svg" %}
  </span>

Out-of-Band Swap

Usar hx-swap-oob para mostrar callouts sin afectar el target principal.

<!-- Formulario que actualiza contenido y muestra callout -->
<form hx-post="/api/update-profile/"
      hx-target="#profile-content">
  ...
</form>

<!-- Contenedor para callouts OOB -->
<div id="notifications"></div>
<!-- Django response con OOB swap -->
<!-- Contenido principal (reemplaza #profile-content) -->
<div id="profile-content">
  <p>Perfil actualizado...</p>
</div>

<!-- Callout OOB (se inserta en #notifications) -->
<div id="notifications" hx-swap-oob="beforeend">
  <div class="ux-callout ux-callout--success ux-callout--dismissible"
       x-data="uxCallout()" x-show="visible" x-transition>
    <span class="ux-callout__icon">...</span>
    <div class="ux-callout__content">
      <p class="ux-callout__text">Perfil actualizado correctamente.</p>
    </div>
    <button class="ux-callout__close" @click="dismiss()">X</button>
  </div>
</div>
