Sistema de Layout

UX utiliza un sistema de layout inspirado en Ionic con contenedores flexibles que permiten crear interfaces de aplicaci√≥n con header, content y footer.

Estructura Basica

La estructura principal consiste en: ux-app > ux-page > ux-header + ux-content + ux-footer

<body class="ux-app">
  <div class="ux-page">
    <header class="ux-header">
      <!-- Toolbar, navbar, etc -->
    </header>

    <main class="ux-content">
      <!-- Contenido scrolleable -->
    </main>

    <footer class="ux-footer">
      <!-- Tabbar, buttons, etc -->
    </footer>
  </div>
</body>

ux-app

Contenedor raiz de la aplicacion. Ocupa toda la pantalla (100dvh) y tiene overflow hidden para que los hijos manejen el scroll.

.ux-app {
  display: flex;
  flex-direction: column;
  min-height: 100dvh;
  max-height: 100dvh;
  overflow: hidden;
  background-color: var(--ux-background);
}

ux-page

Contenedor de pagina con estructura flexible. Soporta header (arriba), content (centro, scrolleable), y footer (abajo).

Transparencia por Defecto

Header, content y footer son transparentes por defecto. Usa variantes para agregar fondo.

Header Transparente

El contenido se ve a traves del fondo porque header, content y footer son transparentes.

Header/Footer Translucido (Glass)

Usa ux-header--translucent o ux-footer--translucent para efecto glass.

Header Translucido

Contenido con tarjeta glass sobre imagen de fondo.

<header class="ux-header ux-header--translucent">...</header>
<footer class="ux-footer ux-footer--translucent">...</footer>

Header/Footer Solido

Usa ux-header--solid o ux-footer--solid para fondo solido.

<header class="ux-header ux-header--solid">...</header>
<footer class="ux-footer ux-footer--solid">...</footer>

Header/Footer Fijo

Usa ux-header--fixed o ux-footer--fixed para fijar al viewport.

<!-- Header fijo al viewport -->
<header class="ux-header ux-header--fixed ux-header--translucent">
  <div class="ux-toolbar">...</div>
</header>

<!-- Content con padding para el header fijo -->
<main class="ux-content ux-content--has-header">
  ...
</main>

<!-- Footer fijo al viewport -->
<footer class="ux-footer ux-footer--fixed ux-footer--safe-area">
  <div class="ux-tabbar">...</div>
</footer>

Safe Area (Notch)

Para dispositivos con notch, usa las clases safe-area.

<!-- Header con safe area para notch -->
<header class="ux-header ux-header--safe-area">...</header>

<!-- Footer con safe area para home indicator -->
<footer class="ux-footer ux-footer--safe-area">...</footer>

<!-- Content con safe area completo -->
<main class="ux-content ux-content--safe-area">...</main>

Variantes de Content

El contenido tiene varias variantes utiles.

<!-- Content con padding -->
<main class="ux-content ux-content--padded">...</main>

<!-- Content sin scroll -->
<main class="ux-content ux-content--no-scroll">...</main>

<!-- Content fullscreen (posicion absoluta) -->
<main class="ux-content ux-content--fullscreen">...</main>
