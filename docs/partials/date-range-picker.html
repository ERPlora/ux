Date Range Picker

Selector de rangos de fechas con presets y calendario dual.

Selector Basico

Usa el componente uxDateRangePicker para seleccionar rangos de fechas.

<div class="ux-date-range"
     :class="{ 'ux-date-range--open': isOpen }"
     x-data="uxDateRangePicker({
       placeholder: 'Seleccionar fechas'
     })">
  <button class="ux-date-range__trigger" @click="toggle()">
    <span x-html="icons.calendar"></span>
    <span x-text="displayText || placeholder"></span>
  </button>
  <!-- Panel content... -->
</div>

Con Fechas Iniciales

Configura fechas iniciales con startDate y endDate.

Calendario Unico

Usa singleCalendar: true para mostrar solo un calendario.

Tamanos

Usa las clases .ux-date-range--sm y .ux-date-range--lg para tamanos diferentes.

Opciones

Opcion Tipo Default Descripcion
placeholderString'Seleccionar fechas'Texto placeholder
formatString'DD/MM/YYYY'Formato de fecha
startDateDatenullFecha inicial de inicio
endDateDatenullFecha inicial de fin
minDateDatenullFecha minima seleccionable
maxDateDatenullFecha maxima seleccionable
showPresetsBooleantrueMostrar presets laterales
presetsArray[...]Array de presets personalizados
singleCalendarBooleanfalseMostrar solo un calendario
autoApplyBooleanfalseAplicar automaticamente al seleccionar

Formato de Presets

Personaliza los presets con un array de objetos.

const presets = [
  {
    label: 'Hoy',
    getValue: () => {
      const d = new Date();
      return [d, d]; // [startDate, endDate]
    }
  },
  {
    label: 'Ultimos 7 dias',
    getValue: () => {
      const end = new Date();
      const start = new Date();
      start.setDate(start.getDate() - 6);
      return [start, end];
    }
  }
];

Eventos

Evento Detalle Descripcion
daterange:open-Panel abierto
daterange:close-Panel cerrado
daterange:change{ startDate, endDate }Rango seleccionado
daterange:clear-Seleccion limpiada

Metodos

Metodo Descripcion
open()Abre el panel
close()Cierra el panel
toggle()Alterna el panel
apply()Aplica la seleccion actual
cancel()Cancela y cierra
clear()Limpia la seleccion
setDates(start, end)Establece las fechas programaticamente
getDates()Obtiene { startDate, endDate }
selectPreset(preset)Aplica un preset

HTMX + Django

Integra el date range picker con HTMX para filtrar datos por rango de fechas en Django.

Filtro de Reportes

Filtra una tabla de reportes por rango de fechas usando el evento daterange:change.

<!-- Filtro con HTMX -->
<div x-data="uxDateRangePicker()"
     @daterange:change.window="
       $refs.startDate.value = $event.detail.startDate.toISOString().split('T')[0];
       $refs.endDate.value = $event.detail.endDate.toISOString().split('T')[0];
       htmx.trigger('#filter-form', 'submit');
     ">
  <div class="ux-date-range" :class="{ 'ux-date-range--open': isOpen }">
    <button class="ux-date-range__trigger" @click="toggle()">
      <span x-html="icons.calendar"></span>
      <span x-text="displayText || 'Filtrar por fecha'"></span>
    </button>
    <!-- Panel... -->
  </div>
</div>

<!-- Formulario oculto para HTMX -->
<form id="filter-form"
      hx-get="/reportes/"
      hx-target="#reports-table"
      hx-trigger="submit"
      hx-indicator="#loading">
  <input type="hidden" name="start_date" x-ref="startDate">
  <input type="hidden" name="end_date" x-ref="endDate">
</form>

<div id="reports-table">
  <!-- Tabla de reportes -->
</div>

Vista Django

Procesa el filtro de fechas en el backend.

# views.py
from django.shortcuts import render
from django.http import HttpResponse
from datetime import datetime
from .models import Report

def reports_list(request):
    reports = Report.objects.all()

    # Filtrar por rango de fechas
    start_date = request.GET.get('start_date')
    end_date = request.GET.get('end_date')

    if start_date and end_date:
        start = datetime.strptime(start_date, '%Y-%m-%d')
        end = datetime.strptime(end_date, '%Y-%m-%d')
        reports = reports.filter(created_at__date__range=[start, end])

    # Si es peticion HTMX, devolver solo la tabla
    if request.headers.get('HX-Request'):
        return render(request, 'partials/reports_table.html', {
            'reports': reports
        })

    return render(request, 'reports/list.html', {
        'reports': reports
    })

Filtro con Presets Personalizados

Define presets especificos para tu aplicacion.

<div x-data="uxDateRangePicker({
       presets: [
         {
           label: 'Hoy',
           getValue: () => {
             const d = new Date();
             return [d, d];
           }
         },
         {
           label: 'Esta semana',
           getValue: () => {
             const end = new Date();
             const start = new Date();
             start.setDate(start.getDate() - start.getDay());
             return [start, end];
           }
         },
         {
           label: 'Este mes',
           getValue: () => {
             const now = new Date();
             const start = new Date(now.getFullYear(), now.getMonth(), 1);
             return [start, now];
           }
         },
         {
           label: 'Ultimo trimestre',
           getValue: () => {
             const end = new Date();
             const start = new Date();
             start.setMonth(start.getMonth() - 3);
             return [start, end];
           }
         }
       ]
     })"
     @daterange:change="
       fetch(`/api/ventas/?start=${$event.detail.startDate.toISOString()}&end=${$event.detail.endDate.toISOString()}`)
         .then(r => r.json())
         .then(data => { ventas = data; })
     ">
  <!-- Trigger y panel... -->
</div>

Dashboard con Multiples Filtros

Combina el date range picker con otros filtros HTMX.

<form hx-get="/dashboard/stats/"
      hx-target="#stats-container"
      hx-trigger="change from:find input, change from:find select, daterange:change from:body">

  <!-- Filtro de categoria -->
  <select name="category" class="ux-select">
    <option value="">Todas las categorias</option>
    <option value="ventas">Ventas</option>
    <option value="inventario">Inventario</option>
  </select>

  <!-- Filtro de sucursal -->
  <select name="branch" class="ux-select">
    <option value="">Todas las sucursales</option>
    {% for branch in branches %}
    <option value="{{ branch.id }}">{{ branch.name }}</option>
    {% endfor %}
  </select>

  <!-- Date Range Picker -->
  <div x-data="uxDateRangePicker({ autoApply: true })"
       @daterange:change="
         document.querySelector('[name=start_date]').value = $event.detail.startDate.toISOString().split('T')[0];
         document.querySelector('[name=end_date]').value = $event.detail.endDate.toISOString().split('T')[0];
       ">
    <div class="ux-date-range" :class="{ 'ux-date-range--open': isOpen }">
      <button type="button" class="ux-date-range__trigger" @click="toggle()">
        <span x-html="icons.calendar"></span>
        <span x-text="displayText || 'Rango de fechas'"></span>
      </button>
      <!-- Panel... -->
    </div>
  </div>

  <input type="hidden" name="start_date">
  <input type="hidden" name="end_date">
</form>

<div id="stats-container">
  {% include 'partials/dashboard_stats.html' %}
</div>

URL Parameters

Filtra usando parametros de URL para compartir enlaces con filtros aplicados.

<div x-data="uxDateRangePicker({
       startDate: new URLSearchParams(location.search).get('start')
         ? new Date(new URLSearchParams(location.search).get('start'))
         : null,
       endDate: new URLSearchParams(location.search).get('end')
         ? new Date(new URLSearchParams(location.search).get('end'))
         : null
     })"
     @daterange:change="
       const url = new URL(location.href);
       url.searchParams.set('start', $event.detail.startDate.toISOString().split('T')[0]);
       url.searchParams.set('end', $event.detail.endDate.toISOString().split('T')[0]);
       history.pushState({}, '', url);
       htmx.ajax('GET', url.pathname + url.search, '#results');
     ">
  <!-- Date Range Picker -->
</div>
